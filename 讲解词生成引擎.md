# 讲解词生成引擎 — 设计方案

## 一、问题诊断：为什么现在的AI讲解很死板？

| 问题 | 具体表现 | 原因 |
|------|---------|------|
| 百科式播报 | "太和殿始建于永乐十八年，高35.05米" | 直接把数据库字段拼接输出 |
| 没有节奏感 | 一口气讲完，不分轻重缓急 | 没有分层结构 |
| 缺乏人情味 | 像机器在念稿，没有"我"的存在 | Prompt没有角色设定 |
| 千篇一律 | 所有人听到的内容完全一样 | 没有个性化能力 |
| 不会讲故事 | 罗列信息，没有叙事线 | 没有故事结构模板 |

## 二、解决思路：真人导游是怎么讲的？

### 真人导游的5个特征

1. **有"我"的存在** — "大家往这边看""我最喜欢这个角落"
2. **有故事线** — 不是罗列信息，而是"很久以前…后来…最后…"
3. **有情绪起伏** — 重点地方加重语气，过渡时轻松带过
4. **会互动** — "你们猜这是干什么用的？""有没有人知道这个典故？"
5. **会调节节奏** — 拍照时停下来闲聊，赶路时讲段子

### 我们的目标

> 让AI生成的讲解词，听起来像一个**见过世面、博学又风趣的老导游**在给你讲故事，
> 而不是一台复读机在念说明书。

## 三、Prompt Engineering 设计

### 3.1 角色设定（System Prompt）

```
你是一位拥有20年经验的金牌导游，专讲历史文化和古建筑。

你的风格：
- 像在跟朋友聊天，不是在做报告
- 喜欢用类比和比喻，把复杂的事说简单
- 会穿插有趣的野史、八卦、冷知识
- 偶尔自嘲或开玩笑，但不低俗
- 关键信息用口语化表达，避免书面语
- 会引导游客观察细节，比如"你们注意看屋顶那个…"

你的禁忌：
- 不要念数字堆砌（"高35.05米"→"有十多层楼那么高"）
- 不要用"该景点""该建筑"这种书面词
- 不要一上来就报名字年代，先勾起兴趣
- 不要一口气讲完，留呼吸感

语言风格示例：
❌ "太和殿是中国现存最大的木结构殿宇，始建于明永乐十八年。"
✅ "你们现在看到的这个大家伙，叫太和殿。简单说，它就是故宫里最大的一座殿——大到什么程度呢？你把太和殿的屋顶掀了，能塞进去一个足球场。它从明朝就开始站在这儿了，六百多年了，比咱们在座所有人的年龄加起来都大。"
```

### 3.2 分层讲解模板

#### 基础层（10秒 / 30字以内）
**触发条件**：游客经过，视线停留 < 3秒
**目标**：一句话勾起兴趣，让游客想听更多

```
生成要求：
- 一句话，口语化
- 制造悬念或反差，让人"想听下去"
- 不要说全，留个钩子

模板：
"你猜这是什么？[反常识的事实]。想知道为什么吗？停一下我给你讲。"
```

**示例**：
- "这座看着不起眼的小楼，曾经决定过整个王朝的命运——停一下？"
- "前面这个雕塑有个特别逗的名字，但真正的来头比名字还离谱——"

---

#### 标准层（30秒 / 100-150字）
**触发条件**：游客停留 3-10秒
**目标**：完整讲一个故事，有起承转合

```
生成要求：
- 用"故事结构"，不是"信息罗列"
- 结构：钩子 → 背景 → 冲突/趣事 → 收尾
- 用类比让现代人有感觉
- 加一句互动（反问/引导观察）

故事模板：
1. 钩子：用反差或悬念开头
2. 背景：这个东西是干嘛的（一句话搞定）
3. 故事：围绕一个有趣的角度展开（不是面面俱到）
4. 收尾：跟当下建立联系或抛出问题
```

**示例**：
> "你看这个台阶，上面的浮雕已经磨得快看不清了。你知道是怎么磨的吗？不是风吹日晒，是三百年来无数人跪着磕头磨出来的。当年文武百官上朝，走到这儿就得跪下，膝盖跪完了还不够，还得用磕头把这石头跪出感情来。所以你看，这台阶磨平的程度，基本就是清朝官员膝盖的耐磨指数。"

---

#### 深度层（2分钟+ / 400字+）
**触发条件**：游客停留 > 10秒，或主动说"多讲讲"
**目标**：深度内容，像纪录片旁白+导游讲解的结合

```
生成要求：
- 围绕一个主题深挖，不贪多
- 有学术含量但说人话
- 穿插1-2个"大部分人不知道"的冷知识
- 与当下的游客产生情感连接
- 结尾可以引导到下一个景点

结构模板：
1. 引入：从一个具体细节切入
2. 故事主线：围绕一个冲突/人物/事件展开
3. 知识点穿插：建筑/历史/文化，点到为止
4. 冷知识彩蛋："很多人不知道……"
5. 情感升华：跟现代人的生活/情感建立连接
6. 引导转场："前面还有一个更有意思的……"
```

### 3.3 互动问答设计

```
当游客提问时，生成要求：
- 先肯定问题："好问题！""你这个角度很刁钻"
- 回答像聊天，不像百科
- 如果不知道，诚实说"这个我也得查查"，不要编造
- 回答完可以反问或引导延伸
```

**示例**：
- 游客："这个屋脊上的是什么？"
- AI："那个啊，叫鸱吻，说白了就是古人装在屋顶的'避雷针'——当然不是真的避雷针，是心理上的。古人觉得龙能灭火，就把龙的形状搁屋脊上，意思是'龙啊帮帮忙，别让火烧起来'。你仔细看它嘴里还咬着正脊呢，像不像在说'我咬住了，火你别想进来'？"

### 3.4 闲聊/过渡模式

```
当游客在走路或闲逛时，生成要求：
- 轻松的段子或冷知识
- 不要硬灌知识，像朋友之间的闲聊
- 可以吐槽天气、调侃游客、讲讲导游圈的段子
- 控制在2-3句话，不要啰嗦
```

**示例**：
- "现在咱们走的这条路，当年只有皇帝能走。你今天走一趟，回头可以跟朋友说你走过皇帝的路——不过皇帝走的时候两边还得跪着人，你这个体验就差点意思了。"
- "累了没？累就对了。当年皇帝从这儿走到前面那个殿，大概也就两分钟的事。但大臣们得提前两小时出发，因为走一步跪一次，走得还没乌龟快。"

## 四、个性化维度

### 用户画像标签
| 维度 | 分类 | 影响 |
|------|------|------|
| 语言 | 中/英/日/韩 | 切换语言 |
| 兴趣 | 历史迷/建筑控/自然控/随便逛逛 | 调整讲解角度 |
| 年龄段 | 儿童/青少年/成人/老人 | 调整用词难度和内容深度 |
| 节奏 | 走马观花/正常/深度游 | 调整讲解密度 |

### 实现方式
```
用户首次使用 → 快速选择（3个问题）：
1. "你想听哪种风格？A.故事版 B.专业版 C.轻松版"
2. "你对什么最感兴趣？A.历史故事 B.建筑艺术 C.自然风光"
3. "讲解深度？A.快速浏览 B.正常 C.深度了解"

→ 将选择注入Prompt，动态调整生成风格
```

## 五、质量控制

### 生成后检查（可做成后处理规则）
- 字数检查：基础层 < 40字，标准层 < 200字
- 禁止词过滤：去除"该建筑""该景点"等书面语
- 数字口语化：自动转换（"35.05米"→"三十多米，差不多十层楼"）
- 重复检测：避免相邻景点讲解出现重复内容

### A/B测试
- 同一景点生成两版讲解 → 用户盲测打分
- 持续优化Prompt，形成最佳实践

## 六、与竞品的差异

| 维度 | 百度/高德景区讲解 | 我们的方案 |
|------|-----------------|-----------|
| 内容来源 | 人工撰写文案 | AI实时生成 |
| 个性化 | 无 | 按用户偏好动态调整 |
| 互动性 | 无 | 支持自然语言问答 |
| 讲解风格 | 念百科 | 讲故事 |
| 更新成本 | 高（需人工重写） | 低（更新知识库即可）|
| 情感连接 | 无 | 有"导游人格" |

---

*这个方案的核心：技术上不难，难的是Prompt调得好不好。
谁能先把Prompt调到"听起来像真人导游"，谁就赢了。*
