## 一、项目概述

### 1.1 一句话定义
基于Rokid AI眼镜，打造"看到即学到"的沉浸式景区智能讲解体验——游客看向哪里，讲解就跟到哪里。

### 1.2 核心理念
- **解放双手**：无需扫码、无需举手机，抬头即见
- **所见即所讲**：AI视觉识别+实时讲解，视线所及之处自动触发内容
- **千人千面**：根据游客兴趣、语言、游览节奏个性化推荐

## 二、痛点分析

### 2.1 传统景区讲解的困境

| 痛点 | 现状 | 我们的解法 |
|------|------|-----------|
| 导游贵且难约 | 旺季200-500元/次，需提前预约 | AI眼镜随戴随用，边际成本趋零 |
| 自助讲解体验差 | 扫码/按键，打断游览节奏 | 无感触发，看什么讲什么 |
| 信息密度不可控 | 同一段讲解对所有人一样 | 根据停留时间、兴趣自动调节深度 |
| 语言障碍 | 多语言导游稀缺 | 实时多语言切换（Gemini多语言能力） |
| 无障碍缺失 | 听障游客几乎无法获取讲解 | AR实时字幕叠加 |

### 3.2 核心功能模块

#### 功能一：智能景点识别
- **技术路径**：摄像头实时采集 → 端侧轻量模型初筛 → 云端精准匹配
- **识别方式**：视觉特征匹配 + GPS辅助定位 + 三维空间特征
- **响应速度**：本地预识别 < 200ms，云端确认 < 1s
- **亮点**：结合华测高精度三维地图数据，实现"空间级"精准定位

#### 功能二：AI实时讲解
- **讲解生成**：基于Gemini大模型，根据景点信息实时生成讲解内容
- **分层讲解**：
  - 基础层：景点基本信息（名称、年代、历史背景）
  - 兴趣层：根据用户停留时长自动深入（停留>10s触发深度讲解）
  - 互动层：支持语音提问（"这个雕塑是谁？"→ 实时回答）
- **讲解风格可选**：专业严谨 / 轻松幽默 / 儿童版 / 故事版

#### 功能三：AR信息叠加
- **景点标注**：视线所及的建筑/景点上方浮现名称标签
- **历史还原**：AR叠加历史照片/复原图（如：废墟→原貌复原）
- **路线指引**：AR箭头指引最优游览路线
- **实时字幕**：讲解内容以AR字幕形式显示在视野下方（无障碍模式）

#### 功能四：多语言无障碍
- **多语言讲解**：中文/英文/日文/韩文一键切换
- **听障模式**：实时语音转AR字幕 + 手语动画指引
- **适老化**：大字体模式、语速调节、一键求助

### 3.3 用户体验流程

```
游客戴上眼镜 → 进入景区
      │
      ▼
系统自动定位 → 识别当前景点
      │
      ▼
AR浮现景点名称 → 自动开始基础讲解
      │
      ├── 游客继续走 → 切换下一景点
      │
      ├── 游客驻足 >10s → 深度讲解展开
      │
      ├── 游客提问 → AI实时回答
      │
      └── 游客说"换个语言" → 切换讲解语言
```

> 核心思路：**AI能力是主角，Rokid眼镜是AI的载体**
> 我们不做底层AR开发，而是调用Rokid现成SDK能力，把精力集中在AI链路上

│  · 调用显示模块实现信息叠加                              │
│  （以上均为SDK调用，无需底层AR开发）                      │
└──────────────────────┬──────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────┐
│                   AI 核心链路（重点）                     │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  ① 视觉识别层                                           │
│  · 景点图像识别（基于预训练模型 / 视觉大模型）           │
│  · 图像→文本描述（视觉理解能力）                        │
│  · GPS辅助定位（粗定位）                                 │

│  · 讲解内容生成（分层：基础/深度/互动）                  │

## 五、应用场景

### 5.1 典型场景一：历史古迹游览
**场景**：游客走进故宫太和殿区域
- 眼镜自动识别太和殿 → AR浮现"太和殿·始建于明永乐十八年"
- 自动开始讲解建筑历史、建筑特色
- 游客抬头看屋脊 → 触发"十大脊兽"专题讲解
- 游客问"这棵树多少年了？"→ AI回答并标注树龄

### 5.2 典型场景二：自然景区导览
**场景**：游客在黄山风景区
- 识别迎客松 → 讲解树龄、生长环境、文化意义
- AR指引最佳拍照点
- 提示"前方200米陡坡，注意安全"
- 根据天气自动调整路线建议

### 5.3 典型场景三：无障碍旅游
**场景**：听障游客参观博物馆
- 讲解内容全部以AR字幕显示
- 重要信息用手语动画辅助
- 触觉震动提示关键节点

## 六、技术亮点（核心差异化）

### 6.1 AI多模态理解
> 我们的核心壁垒——不只是"识别景点"，而是"理解场景"

- **视觉+语言多模态**：眼镜拍到什么，Gemini就能讲什么，无需预先标注
- **开放式问答**：游客问"这棵树多少年了？"→ AI结合图像+知识库实时回答
- **上下文感知**：记住游客已看过什么，避免重复讲解，智能推荐下一步

### 6.2 智能讲解引擎
> 分层生成，千人千面

- **三级讲解深度**：
  - 基础层（10秒）：名称+年代+一句话亮点
  - 标准层（30秒）：历史背景+建筑特色+趣味故事
  - 深度层（2分钟+）：学术级讲解+关联景点+文化延伸
- **自适应触发**：根据游客停留时长自动调节讲解深度
- **个性化推荐**：根据用户兴趣（历史/建筑/自然）调整讲解风格

### 6.3 地信背景赋能
> GIS能力不是开发AR，而是让AI更"懂空间"

- **视觉定位辅助**：用图像特征匹配辅助GPS，提升定位准确率
- **景点知识图谱**：基于GIS空间关系构建景点关联网络
- **空间语义理解**：知道游客"面朝什么方向""在看哪个建筑"

### 6.4 多语言无障碍
- **实时多语言切换**：Gemini原生多语言能力，一键切换讲解语言
- **听障模式**：语音→实时字幕叠加在眼镜显示上
- **适老化**：语速可调、大字显示、一键求助

## 八、竞争优势总结

1. **AI能力突出**：多模态大模型 + 分层讲解引擎，不是简单的"语音播放器"
2. **体验差异化**：所见即所讲 + 自然语言问答，比扫码讲解体验跃升一个时代
3. **地信背景加持**：空间语义理解让AI更"懂"景区，识别更准
4. **社会价值**：无障碍旅游模式 → 评委加分项
5. **平台契合**：充分释放Rokid AI眼镜的AI+视觉能力
